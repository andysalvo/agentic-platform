Validate MCP Tool Definitions

Free validator for MCP tool quality

Validate your MCP tool definitions for free using mcp_manifest_lint, the only dedicated MCP tool validator. It checks descriptions, parameter schemas, naming conventions, annotations, and anti-patterns that cause agents to misuse or skip your tools.

MCP Endpoint https://api.asalvocreative.com/mcp

Step-by-Step Validation

1. Connect to the MCP server

Add the Agentic Platform to your MCP client. For Claude Code:

claude mcp add agentic-platform --transport http https://api.asalvocreative.com/mcp

2. Prepare your tool definitions

Gather your MCP tool definitions as a JSON array. Each tool object should include name, description, and inputSchema.

3. Run the validator

Call mcp_manifest_lint with your tool definitions:

mcp_manifest_lint({
  tools_json: '[{"name": "search_docs", "description": "Search documentation", "inputSchema": {"type": "object", "properties": {"query": {"type": "string"}}}}]'
})

4. Review the report

The validator returns a structured report with pass/fail status and specific fixes for each issue:

MCP Manifest Lint Report
========================

Tool: search_docs
  PASS  Description present and clear
  WARN  Parameter "query" missing description
  WARN  Missing annotations (readOnlyHint, destructiveHint)

1 tool checked. 0 errors, 2 warnings.

What Gets Validated

  • Descriptions -- missing, empty, or too vague for an agent to understand the tool's purpose
  • Parameter schemas -- missing types, missing descriptions, missing constraints on string/number fields
  • Naming conventions -- inconsistent casing, ambiguous names like "do_stuff" or "process"
  • Annotations -- missing readOnlyHint, destructiveHint, and other MCP annotations
  • Scope -- overly broad parameter definitions that give agents too much freedom
  • Agent confusion -- definitions that would cause an agent to select the wrong tool or pass bad arguments

Why Validate Before Publishing

MCP registries like Smithery and Glama surface tools based on definition quality. Poor descriptions and missing schemas mean agents skip your tools entirely.

Running the validator before publishing catches the issues that matter. One pass through mcp_manifest_lint takes seconds and prevents hours of debugging agent behavior.
