Why Per-Token Pricing Is Misleading

Agentic workloads are not single API calls. An agent making 8 tool calls sends the full conversation history each turn. Your actual cost is driven by context accumulation, not the sticker price per token.

A "cheap" model that takes 15 turns to complete a task can cost more than a "premium" model that finishes in 3. The estimate_agent_cost tool accounts for this by modeling the full agent loop.

Estimate Your Actual Cost

Call estimate_agent_cost with your model, context size, and expected tool calls:

estimate_agent_cost({
  model: "claude-sonnet",
  avg_input_tokens: 4000,
  avg_output_tokens: 800,
  tool_calls_per_run: 6,
  runs_per_day: 100
})

It returns a cost breakdown per run, per day, and per month -- with a comparison across model tiers so you can see exactly where the tradeoffs are.
