AI Agent Governance Framework

Safety checklist and governance scoring for autonomous agents

Score your AI agent's governance posture using agent_health_check, a free tool that evaluates your agent against a structured safety checklist. It covers authentication, rate limiting, error handling, logging, human-in-the-loop controls, and scope constraints. No API key required.

The Governance Checklist

Authentication and Authorization

Every external API call and tool invocation should be authenticated. Agents should operate with least-privilege credentials scoped to their specific task.

Rate Limiting

Uncapped agents can burn through API budgets in minutes. Rate limits should exist at both the agent loop level (max iterations) and the API call level (requests per minute).

Error Handling

Agents need structured error handling with graceful degradation. A failed tool call should not crash the loop or trigger infinite retries.

Logging and Audit Trail

Every decision, tool call, and outcome should be logged. Without an audit trail, you cannot debug agent behavior or prove compliance.

Human-in-the-Loop Gates

Destructive actions (deleting data, sending money, publishing content) should require human approval. The agent should pause and surface the decision, not proceed autonomously.

Scope Constraints

Agents should have explicit boundaries on what they can access and modify. An agent scoped to "read and summarize" should not have write permissions.

Fallback Behavior

When a tool or API is unavailable, the agent should have a defined fallback. Silently dropping failures leads to incorrect results.

Cost Controls

Set per-run and daily spending caps. An agent that enters a reasoning loop with a frontier model can generate hundreds of dollars in API costs within an hour.

How Governance Scoring Works

The agent_health_check tool evaluates your agent across each checklist category. Describe your agent's architecture, and it returns a score with specific findings.

agent_health_check({
  agent_description: "A customer support agent that reads tickets from Zendesk, drafts responses using Claude Sonnet, and posts replies. It runs every 5 minutes via cron. Auth via API keys stored in environment variables. No rate limiting configured. Logs to stdout.",
  check_type: "governance"
})

The report highlights what you have in place, what is missing, and what to fix first. It prioritizes by risk -- missing auth and missing cost controls rank higher than missing logging.
